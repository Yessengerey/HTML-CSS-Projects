Basics of Web Content

In general, all web documents use Hypertext Mark-up Language (HTML). HTML is the fundamental basis for all the content that we view on various web-sites everyday. Of course there are other embedded technologies, however they do not build up the content, they enhance its display and content.

Rendering process

In order for the content of a web page to be visible in a browser, the information for that content goes through a two step process:

  1. Firstly, the source code from a flat file via either HTTP or a local file system is delivered to the browser. That source code consists of mainly just the main information and other tags which will be used for the second step.
  2. The web-browser then processes the document and displays it to the user of that browser. And so the appearance of the content is built up from a combination of source construction, which is made up by the author of the content, and rendering capability of the web-browser itself (basically things that the browser is able to show).

Three Concerns

A proper web document typically has three factors that it's mostly concerned about:

Content Organization

The content is the information carried out by the page. It is very important for the content to have a meaningful structure which includes section headings, proper paragraphs , various tags, and etc. Proper structure of the content ensures proper understanding of the information by the user.

Presentation

The "Eye-candy" for the users. This factor includes the basic visual presentation of the web page which includes colour schemes and graphics, typefaces, layouts, generally anything that makes it look rather visually attractive will be in this concern.

Behaviour

In order to make the web page be even more visually attractive, some behaviour can be added, which includes interactive or responsive functionality. For example, a functionality that allows the web page to respond to certain user inputs. Another functionality could be enabling the browser to handle certain inconsistencies or requirements, like detecting the available scree size. The "Eye-candy" here is based on the activity going on on the web page.
