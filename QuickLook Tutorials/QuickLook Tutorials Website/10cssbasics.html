Basics of CSS

CSS stands for Cascading Style Sheets and as was mentioned in the introductory section of this tutorial, CSS is all about style and visual presentation that is applied to the structured mark-up of the web document. CSS allows a separation of style from structure and it has full control over almost every element in the web page.

But why do we need CSS? Well the answer is pretty obvious - plain black-on-white pages are dull and visually unattractive. So CSS allows you to express you creativity!

However, there is still a trick to styling up your page. "Overstyling" may lead to loss of information transmission to the user and "Understyling" may just not be enough for the user to be interested in reading the information.

Formatting Rules and Syntax

CSS has a set of certain rule that must be followed in order to open the full potential of the styling. The rules basically consist of Selectors which are simply the Tags (p, div, ul, etc.) and Declarators that are added in the CSS style sheet (e.g. background, color, etc.).

p {background-color: blue;}

The above is an example of a style rule which include a Selector (p) and a declaration that is enclosed in curly brackets with a format of {style-property: value}. If there are more than one declaration, then they can be separated by a semi-colon.

ul {margin-right: 10%; font-size: 10px}

In case you want to apply some style rules to several selectors at once, the selectors can be separated by a comma

h1,h2,h3 {background-color: red; font-style: italic;}

Setting Style Rules

There are two by which the style rules can be set - Internal, External and/or Inline.

Internal

Internal style rules must be set inside of the head section of the HTML.

<head>
<title>A Title<title>
<style type="text/css">
p {background-color: blue; font-style: italic;}
</style>
 </head>

External

External style rules are set from a separate file and are linked to the web document. The style sheet is added through either a self closing link or though an @import, both ways have to be added into the <head> section of the web document. Moreover multiple style sheets can be linked to a single web document.

<head>
<title>A Title</title>
<link rel="stylesheet" type="text/css" href="style_file.css" />
 </head>

<head>
<title>A Title</title>
<style type="text/css">
@import url("style_file.css")
</style>
 </head>

Inline

Inline style rules are set inside of the opening of any tag. The CSS rules are put as values for the style attributes.

<p style="color: whie; background-color: green;">line<p>

Selectors

There are three main selector types used for the style rules to be applied in the HTML: Tags - generally any tags can be used; Class - match all tags that contain that specific class; Id - tag containing an id is treated as a unique tag as that id cannot be used for another tag. These tags can be combined together in various combinations in order to make the application of style rule more specific.

Class

Class is a very general selector as multiple tags can be assigned the same class attribute and so any set of style rules will be applied to all elements with that tag containing that class. Class selectors are set inside of an opening tag in this format:

<tag class="class name">...content...</tag>

Then in the CSS style sheet rules are set in a specific form. The dot (.) specifies the class and anything before that dot specifies the tag in which that class was written.

.class_name {style-property: value;}

tag.class_name {style-property: value;}

Id

These selectors are used in order to make a certain element unique in the page. An id can only be used only once per page, however the same Id can be used on multiple pages if you'd like so

<tag id="id_name">...content...</tag>

In order to apply some rules in the Style sheet to that specific id, a hash tag (#) is used in the CSS selector pattern

#id_name {style-property: value;}
