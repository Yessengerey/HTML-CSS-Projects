Tables and Lists

Tables

Tables were introduced into HTML in order to help the representation of certain data like statistics of some type, in a tabular and neat way. Tables can also be used for the layout of the page's content, however that should be done with care. Tags and Attributes can also be used when creating tables, though they would only be able to provide just basic structure and representation. Other more visually attractive enhancements can and are suggested to be made through CSS.

Basic structure

  • Table should open up with a simple tag <table>. Since the table tag is of a paired type, it should also be closed properly.
  • The content that is intended to be a header in the table should be enclosed in <th> tag; and the content that is not a heading should be enclosed in <td>
  • Cells and headers themselves have to be within a row for the table consequently they have to be enclosed by <tr> tag.
  • Title of the table can either be written via <p> or <caption> tags. Although only through the second tag will the browser actually associate the table with that name. With <p> tag, the browser reads the title of the table only as a 'sentence' above the table

<caption>Title of the table</caption>
<table>
<tr>
<th>Header 1</th><th>Header 2</th>
</tr>
<tr>
<td>Content under header 1</td><td>Content under header 2</td>
</tr>
</table>

Table Attributes

There are also other attributes that enhance the visual representation of the table to a certain extend. Other attributes like colour, are now only to be applied through CSS. However, the examples given below may only be used in HTML5 as XHTML 1.0 nor HTML 4.01 does not allow even those attributes to be applied to the table the Mark-up language.

AttributesExplanation
borderAffects that visibility and thickness of the table's border
widthAffects the width of the table, units of which can either be Numbers (pixels) or percentage
cellspacingAffects the distance between the cells within the table; units are pixels
cellpaddingAffects the distance between the content of the cells and the cells' frames; units are pixels

Lists

Lists can be used to improve the organization of the content as well as simplify the understanding of the presented information for the users. They are also the fundamental basis for dynamic menus and tab effects (with the help of CSS and other embedded technologies). Nevertheless lists' main focus drops on the organization rather than the visual representation. Overall, there are three types of lists: Unordered which is basically bullet points; Ordered that is a numbered list; and Definition - list consisting of terms and their definitions.

Unordered

<ul>
<li>...a line of the list...</li>
<li>...a line of the list...</li>
</ul>

  • All bullet lists open up with the <ul> tag and close with the appropriate one - </ul>.
  • Every line of the lists must be enclosed with the paired <li> tag.

Ordered

<ol>
<li>...a line of the list...</li>
<li>...a line of the list...</li>
</ol>

  1. All ordered (numbered) lists should be enclosed by <ol>...the list...</ol>
  2. Every line of an ordered list has to be wrapped up by <li>...a line of the list...</li>

Definition list

<dl>
<dt>Term A</dt>
<dd>Definition of the Term A</dd>
<dt>Term B</dt>
<dd>Definition of the Term B</dd>
</dl>

The Definition List must be enclosed by <dl>...</dl> tags. The term that is to be defined should be wrapped up by <dt>...</dt> tags and finally the definition of the specified term has to be wrapped by <dd> tag. The end result is shown below:
