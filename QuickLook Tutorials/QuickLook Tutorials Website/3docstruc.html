Structure of the Document

Importance

Unfortunately web pages printed out do not look the way we intend them to. Printed out pages most often do not look the way they look in a web browser. This means that we don't have a full control over the final output. So the fundamental basis of the document structure has to be properly written in order for the final output, or in case a web browser does not read additional embedded technologies, in order for the document to be used by users.

Moreover, properly structured HTML will make your life much easier when it comes to editing and maintaining the documents. It also helps the rendering in browsers to be more consistent as well as users have a possibility to use the accessibility features for display to the fullest.

Basic Structure

An HTML document is composed of three different parts:

  1. A line containing HTML version information

    • <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN"
      "http://www.w3.org/TR/html4/strict.dtd">
    • The above is the Document Type Definition (DTD) declaration which identifies the version of the web document to the browser. The PUBLIC part identifies the version of the mark-up language that the web document is written in (e.g. HTML 1.0, HTML5, etc). However the URL part is for the location of the mark-up specifications and standards.

  2. Declarative header section

    • <head>
      <title>Title of the web document</title>
      <meta http-equiv="content-type" content="text/html;charset=utf-8">
      <head>
    • The 'header' section of the web document is read by the browser, however it is not being shown on the actual page. That section has no content; it only shows the information about the web document that helps the browser to process it. The 'header' must also include the title (<title>) which is shown on the Tab of the page in the web browser. If you do not give a title in the header, the browser will give it a default name, such as "Untitled Document" if you were using an application to create the web-page, or if you were using something like Notepad++, the title will be the name of the web document that you gave when you had created it.
    • Optional declarations in the 'header' section may include additional Metadata (<meta>), Scripts (<script>) or Style-sheets either inside of the actual web document using <style> or through a link to an external document - <link>.

  3. Body

    • <body>
      <div><img src="title_of_the_picture.jpg" alt="alternative title"></div>
      <h1>Head title in the document<h1>
      <p>A paragraph in the main body of the web document which is visible on the actual web page.<br>
      As long as the paragraphs are properly enclosed by <p>...</p>, you can add as many paragraphs as you like
      </p></body>
    • The above section that is enclosed in the <body> is the content that is being processed into the web browser and then shown on the web page. However the visual presentation of the final output might be influenced depending on the additional declarations and references mentioned in the 'header' section of the document.

Furthermore, sections 2 and 3 must be enclosed into <html> tag in order for the web document to be complete. So the final output should, or rather must look somewhat like this:

  • In case of HTML 4.01

    • <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN"
      "http://www.w3.org/TR/html4/strict.dtd">

      <html>

      <head>
      <title>Title of the web document</title>
      <meta http-equiv="content-type" content="text/html;charset=utf-8">
      </head>

      <body>
      .
      .
      .
      </body>

      </html>

  • In case of XHTML 1.0

    • The main difference between HTML 4.01 and XHTML 1.0 is the DTD and the syntax in writing. Instead of writing DTD HTML 4.01, now we have to change it to DTD XHTML 1.0 Strict along with the location of the mark-up specifications. In terms of syntax, in XHTML the non-paired tags have to be self-closing. In addition, an extra attribute in the html tag has to be added as it can be seen in the example below.
    • <!DOCTYPE HTML PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN"
      "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">

      <html xmlns="http://w3.org/1999/xhtml">

      <head>
      <title>Title of the web document</title>
      <meta http-equiv="content-type" content="text/html;charset=utf-8" />
      </head>

      <body>
      .
      .
      .
      </body>

      </html>

  • In case of HTML5

    • HTML5 differs from the older versions only by the simplified DOCTYPE and the meta tag in the 'header'. However, in terms of Syntax either HTML 4.01 or/and XHTML 1.0 can be used.
    • <!DOCTYPE html>

      <html>

      <head>
      <title>Title of the web document</title>
      <meta charset="utf-8">
      </head>

      <body>
      .
      .
      .
      </body>

      </html>
