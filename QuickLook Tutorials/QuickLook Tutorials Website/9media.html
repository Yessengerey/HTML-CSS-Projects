Multimedia

Basics

Non-textual content of any page requires the browser to be able to handle and support them i.e. make them visible and some times support interactivity if needed. In addition there is a certain way for different media objects to be embedded in the web pages, all of which will be discussed in this section of the tutorial.

Nowadays some basic media objects will have a native support from any browser, meaning no extra software is required for the objects to be visible. Such objects would include some simple image formats, perhaps some audio and/or video objects.

However some more complicated types of objects will need some help from certain plug-in applications. These kind of objects' formats would include Flash Player, QuickTime, Media Player, etc.

Adding Images

There is a simple way of adding images to a web page, example of which you can see below:

<img src="name_of_the_image.png" alt="Alt. Name" />

Image should be added through a self-closing tag 'img'. The 'src' attribute is for specifying the location of the image file, should it be from a local file or online in web. The 'alt' attribute's role is to provide an alternative name that will appear instead of the image in case the image does not show up or load in the web page.

There are three main image file types that should be used in order to embed the images into the web page: GIF, JPEG and PNG.

GIF is good for solid graphics as it has only 256 colour palettes. It can also have a transparent background and can be used for simple short animations. Moreover it does not lose anything if the image is being compressed.

PNG is loss-less when compressed as well, it is considered to be a replacement, or some might even say, an improvement for GIF as it not only can have transparency but also can handle higher resolution images.

JPEG is best for photos due to 16 million colour palettes. However at the same time it loses quality if compressed to a smaller file size. When compressed, JPEG introduces blur and so the sharpness of colours and edges get worse.

In general, I would my personal favourite is PNG. As it easily handles high resolution images with full palette and does not lose sharpness if the file is compressed. Although when it comes to dealing with high resolution photos that include very complex colours and their mixes, the best format would be JPEG (if used carefully).

Attributes for an Image

CodesCharacters
srcThe attribute is the location of the image file, either through the local file system or online, in which case you'd have to used the full absolute URL of the source.
altAn alternative name/description of the image used by screen readers or if the image doest not load in the page.
width/heightThe dimensions of the space that browser should reserve for that image, which means change in this attribute does not the alter the size of the actual image; unit - pixels.
titleDescriptive title that can only be seen when you hover the mouse over that image.

Adding other media objects

When we are talking about other media objects its usually either Videos and Audio objects. Most of the formats for those kind of objects are usually supported by the majority of the browsers. Unlike image objects, there are several ways by which video and audio objects can added to the page - <applet> (Java applets; part of HTML spec), <embed> (Netscape; supported by many browsers but never adopted in standard HTML versions), <object> (introduced by W3C into HTML 4.01), <iframe> and <video>/<audio>(part of HTML5). Nevertheless the most popular of all of these methods are <embed> and <object>

Examples given below are some templates of how a video should be added into a web page via various methods.

<embed>

'Embed' is the most widely used method to add media objects like videos to a web page. This is due to the method being supported by most of the browsers of the present and the past. However, YouTube is starting to phase <embed> out and use <iframe> instead.

Below is a template of how an embeding a video object into a web page should look like:

<embed src="location of the video"
type="application/x-shockwave-flash"
width="xxx" height="xxx"
allowscriptaccess="always"
allowfullscreen="true">
</embed>

  • src attribute should specify the location of the video like the full absolute URL
  • type is responsible for specifying the application that the video is run with
  • width and height are the dimensions for the browser window that is set for the video object; the dimensions do not alter the size of the actual video
  • allowscriptaccess allows the video to be affected by scripting
  • allowfullscreen allows the video to be open on full screen

<object>

<object> is the current and the "correct" way of adding an object like a video into a web page. This method was introduced by W3C into HTML 4.01. It is supported by all web browsers that we use today. The only difference is the addition of the parameters <param>. A template for this method can be seen below

<object width="xxx" height="xxx"
type="application/x-shockwave-flash"
data="location of the video">
<param name="movie" value="location of the video" />
<param name="allowFullScreen" value="true"/>
<param name="allowscriptaccess" value="always"/>
</object>

  • type is responsible for specifying the application that the video is run with. This is required by Internet Explorer
  • Instead of src we have to put data to specify the location of the video from which the video should be imported into the web page. Data is required by Firefox
  • Parameters param are the information about the video for the screen reader

<iframe>

Possibly the least complicated and my personal favourite way of adding a video into a web page, however this method may not be supported by all web browsers.

<iframe
width="xxx" height="xxx"
src="location of the video"
frameborder="0"
allowfullscreen>
</iframe>

<video>

An HTML5 version of the method to add a video into a web page. Here controls are added if the web browser is capable of handling it.

<video
width="xxx" height="xxx"
src="location of the video"
type="application/x-shockwave-flash"
controls="controls">
</video>>
