Separation of Concerns

Content Organization (HTML)

The Content Organization is made up of HTML. It is, or rather should be made up of three things: Content, Tags and Attributes.

  • Content
    • The content of the web page itself is just the text and the images. This is the actual information that we read and see on the web page.
  • Tags
    • The tags help to build up the core structure of the displayed content, as tags focus on the semantics of the web page. Tags also have their own default conventions for rendering a functional display. Examples of tags include tags such as for paragraphs: <p>...<p>, or headings: <h1>...<h1>, etc.
  • Attributes
    • Attributes complement tags, or rather they make the content enclosed by those tags more specialized. Attributes are used to give additional properties to specific parts of the document like words, sentences, pictures. Examples include an "id" inside of a <div> tag, an "attribute" inside of a tag, etc.
    • Tags and attributes are explained more in detail further on in this tutorial under a heading "Tags and Attributes", which can be accessed from the main page

NOTE: The information provided in this tutorial is mainly based on HTML 4.01 and XHTML 1.0, as these are the basics of the HTML5. However some references to HTML5 will be made through out this tutorial

Presentation (CSS)

CSS is all about style and visual presentation that is applied to the structured mark-up of the web document. With the help of specific attributes the style can be applied to specific parts of the content; the process is called pattern matching syntax.

All attributes must have a value. The Value syntax, the actual value used with the attributes like a the "id", on the other hand specifies what to apply to the attributed parts of the documented.

CSS allows re-positioning of content as well as it can add external multimedia and transformations.

Behaviour

If we're talking about behaviour - its scripting and interactivity which includes programmes that come along with the web content (as part of that page or linked to it), as well as programs that are compiled and executed on client-side (e.g. Java-script).

Client-side scripting, with the help of Document Object Model (DOM), can interact with the web page and read, write and manipulate certain HTML elements, CSS rules and Client features. This opens up a possibility for, far more rich interaction as well as a dynamic functionality of the web page.
