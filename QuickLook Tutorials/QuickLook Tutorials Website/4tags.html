Tags

As was mentioned before in the introductory part of this tutorial, the tags help to build up the core structure of the displayed content, as tags focus on the semantics of the web page. Tags also have their own default conventions for rendering a functional display. Tags in general are broken up into three different kinds: Paired, Unpaired, Inline and Block level tags.

Paired Tags
Tags that are paired up usually wrap a certain content that is being processed, onto the screen when the web page is open, for example:

<h1>A Heading</h1>

<p>A paragraph</p>

Unpaired Tags
Unpaired tags are obviously those that should be self-closed and that do not wrap any content. However these tags may still have some specific attributes like an 'id' or a 'class'. Examples of unpaired tags are as follows:

<br />

<hr />

<hr id="rule" />

Block Level Tags
Block level tags are there for providing the general structure of the content for the web page. The structure which is built from these tags includes Headings, Paragraphs and Horizontal Rules, which are for the text structure; it also includes the organization of Lists and Tables, which are going to be discussed in detail later on in this tutorial.

Headings are by far the most important components when it comes to building the structure of the web page. They help the users to scan through the web page and understand its general content. Headings can be of 6 different levels (h1 - h6); level one being the biggest and the main heading, which means there can only be one 'h1' heading per page. In addition, all headings are treated as separate blocks, so they cannot and must not be nested within each other.

<h1>Main Heading<h1>

<h2>Sub Heading<h2>

Paragraphs, as was mentioned before, act as a container that encloses the main information that is displayed on the web page via <p>...</p>.

Breaks and Rules do not enclose anything inside of them. However they still do affect the visual representation of the content. Breaks force lines within a, for example paragraph to break. This is not an ideal practise so its better to avoid Breaks as much as possible. Rules on the other hand create lines with which the content can be separated into sections

<br /> - Line break

<hr /> - Horizontal rule

Inline Level Tags
Inline tags have to be nested inside of block level tags. Inline tags are usually used to apply some behaviour to only a specific word/sentence in a paragraph or to a particular part of the web document.

Usually only the Logical types of the inline tags are used. These add an extra rendering intent to the enclosed content. However, you should always keep in mind that different web browsers may render the inline tags differently.

Common logical inline tags include:

TagsExplanation
<em>Enclosed content appears in italics
<strong>Enclosed content appears in bold
<cite>Encloses a citation or a reference
<abbr>Encloses an abbreviation; tag also requires a title attribute
<a href="url">Creates a link from the enclosed content

Another type of inline tags is Physical. However they've only been used in the past. They also produce only a visual effect that can be seen when the web document is rendered by the browser. Right now, it is insisted to use the alternative to the various inline physical tags, which is CSS.

Additional information about Physical inline tags can be found here:
Physical Style Elements - VirginiaTech, Digital Library and Archives

HTML5 Elements

When HTML had reached its 5th version, Flow and Phrasing elements were introduced in order to reflect a logical structure for the more modern we documents. It had also reduced the requirement for the extra id and class attributes, which made 'styling' of the document easier.

Notable additions for Block Level tags were <section>, <nav>, <article>, and for the Inline Level tags <canvas>, <progress>, <output>.

The full list of Tags of various levels that are used in HTML 4.01, XHTML 1.0 and HTML 5 can be seen here
