CSS Positioning

Good CSS cannot be achieved without proper HTML structure. So I'd even say that you have to keep in mind that you basically start writing CSS when you are building up the structure of HTML. The first 'step' to "sensei-ing" CSS is to understand the logic behind the CSS Box model and the Natural Page Flow (source order display of (X)HTML elements). Once you start to view the content of a web page as simply boxes of content, positioning and layout with CSS becomes very easy.

Floating

In order to explain Floating of objects through CSS lets consider a real life example. Let say we have three boxes of some content.

<tag id="box1">...content...</tag>
<tag id="box2">...content...</tag>
<tag id="box3">...content...</tag>

All of them have certain and the same width and height. Let's change the width of the first box, say make it smaller, and float it to the left.

#box1 {width: 200px; float: left;} - What happens next?

  1. The box that had been assigned to float to the left will firstly be pushed to the top as far as possible, over the content of other boxes and to the opposite direction of the specified flow.
  2. Then the box is pushed in the direction of the float as far as possible
  3. As the floated box is now over the content of the other boxes, that content is allowed to wrap around the floated box. However the shape of boxes that remain beneath the box1 do not change the shape.

Boxes that remain beneath another box after that box was floated can be pushed out from beneath the covering box, which is achieved by #box3 {clear: right/left/both;}. The values specify which side of the box should be cleared of floating elements.

Sometimes in order to tackle the problem with the content wrapping around the floated box, you could use columns. Which are simply created by padding the content of the boxes that are beneath the floated box as well as altering the margins in order to move the borders of the boxes as well.

Position Properties

ValueExplanation
staticThis value does not have a position and is set as a default
relativeAllows the element to be positioned 'relative' to the other elements, meaning change in the positioning of other elements may have an effect on the position of that element
absoluteWith this value the element is being removed from the page flow and then be put anywhere you like
fixedThe element is being removed from the page flow and then fixed at the browser viewport, which allows the other content to be scrolled behind that element
