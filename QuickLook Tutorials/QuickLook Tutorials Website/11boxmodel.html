CSS Box Model

Box model is one of the core elements of CSS layout, in fact every element in the page is actually represented as a box, which means that every single tag has a box associated with it

Boxes themselves have certain attributes in terms of "layers" around the text content which they enclose: Border, Padding and Margin.

Border

Border is the outline of the box which itself has three different style attributes:

  • Width - a unit of measurement, e.g. "border-width: 15px;"
  • Style - special style of the actual border can be set through "border-style: solid/dashed/etc."
  • Color - simply give a color to the border of the box, "border-color: red;".

Padding

Padding creates space between the actual content and the border of the box. Usually images are used to fill in the space of the padding. Padding is declared in the way shown below:

  • padding: 20px - all sides acquire padding of 20 px
  • padding-top, padding-left - padding is applied to only one of the sides of the box

Margin

Margin sets the spacing between the outside wall of the border of the box and the boxes of other page elements, which is very useful for managing the content.

  • margin: 20px - all sides acquire margin of 20 px
  • margin-top, margin-left - margin is applied to only one of the sides of the box

Shorthand Declarations

Shorthand declarations are simply declarations that are merged together. They are more precise and much easier to implement. It is suggested to use these declarations as much as possible in order to reduce the amount of the styling code. Some examples of Shorthand Declarations are given below:

Shorthand DeclarationsExplanation
border-width: 10px;Width of all four sides is set to 10px
margin: 10px 20px 5px 25px;Setting margin for top/right/bottom/left
border: 15px dashed red;Specified border is set to all four sides of the box

Width and Height

  • #box {width 300px;}
  • #box {min-width: 450px; max-width: 700px;}

These properties are used in order to impose a defined value on the width and height of the content inside of the box.

Background

The background property simply sets any background filling for the visible space inside of the box and behind the content

  • background-color: xxx; - sets a valid solid/transparent color as the background via either a name or a numeric value
  • background-image: url(location of the img file url/local file access) - sets a chosen image as the background
  • background-position: top/right/left/bottom or top right, etc. - positions the original copy of the image inside of the box
  • background-repeat: repeat/no-repeat - controls the repetition of the image (image is repeated by default)

Images that are added through CSS should only be of decorative intention. Photographs/pictures/images that that are meant to deliver some sort of content should be added to the page via HTML. In addition, only CSS3 multiple images are allowed to be present in a single box. In previous versions of CSS that is not possible.

Content Overflow

Shrinking a box might affect the content in a way that it would simply overflow the frame of the box due to lack of space. And there are several ways of dealing with such kind of problem:

  • overflow: visible; - lets the text to 'spill' out of the box (which is also a default option)
  • overflow: hidden; - hides the overflow meaning the only content that will be visible is that one that fits
  • overflow: scroll/auto; - adds a scrollbar on the side of the inner border of the box via which you can see the content that was supposed to 'spill' out

Display and Visibility

Display property has full control of how an element should be displayed in the flow of the page structure or should it even be displayed at all

  • selector {display: block;} - displays all elements within the specified selector as a block, i.e. one under another.
  • selector {display: inline;} - displays all elements within the specified selector in a line, i.e. one after another.
  • selector {display: none;} - removes elements within the specified selector from the page flow, i.e. elements are not only no longer visible, but the spaces reserved for them by the browser are also removed.

Visibility enables/disables visibility of elements yet not removing them from the page flow, meaning that the space reserved for the elements by the browser can still be seen

  • selector {visibility: hidden;} - hides all elements within the specified selector yet not removing them from the page flow
  • selector {visibility: visible;} - makes all elements within the specified selector (browser default)
  • selector {visibility: collapse;} - removes a row or column from a table leaving s the space for other content to be added
